2. :focus-visible

  • Focus styles highlight an interactive element (like a button, link, or input) when it's "in focus."
  • Typically triggered by:
    • Mouse click
    • Keyboard navigation (usually by pressing the Tab key)
  • Focus styles are essential for accessibility, especially for keyboard users.

Regular Focus

  • The :focus pseudo-class applies whenever an element gains focus, whether by mouse, keyboard, or touch.
  • Issue: This can result in unwanted focus outlines when users interact with elements using a mouse or touch, which often isn't necessary.

When clicking this button, the focus outline appears even if the user is using a mouse.

Focus-visible

  • The :focus-visible pseudo-class is a more refined approach introduced in the CSS Pseudo-Classes Level 4.
  • Purpose: It applies focus styles only when it is appropriate, typically when the user is navigating via keyboard or other non-pointer inputs.
